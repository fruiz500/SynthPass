What is SynthPass?

SynthPass doesn't store your passwords; it synthesizes them on the fly using a Master Key (MK) and the website's name. This means there is no central database to hack and no subscription to pay.

The Golden Rule: Everything depends on your Master Key. It is never stored by the extension. If you lose it, you cannot recover your synthesized passwords. Use the 4-letter Hashili word (e.g., "Bana") that appears as you type to verify your MK is correct without revealing it.

How do I Change a Password?

Rotating a password is a four-step "copy-and-fill" process:

  1. Navigate to the "Change Password" page of the website.
  2. Copy Old Pwd: Without changing your settings, click the Change button in SynthPass. This recreates your current password and copies it to your clipboard.
  3. Generate New Pwd: Change your Serial or constraints, then click Synthesize. All password fields on the page will be filled with the new version.
  4. Authorize: Paste the old password from your clipboard into the "Old Password" field on the website.

The Vault & Secure Notes

Use the Vault for passwords you didn't synthesize or for secure notes.

  • MASTER KEY REQUIRED: You must enter your Master Key to encrypt, decrypt, or delete Vault items.
  • Encrypted Storage: Data is encrypted locally using NaCl cryptography before being saved to browser sync storage.
  • Visual Indicator: The Vault button highlights if a record exists for the current site.
  • Universal Use: If a page has no password fields, decrypting a Vault item will copy it to your clipboard for one minute.
  • To Delete: Click Vault and type DELETE in the prompt.

Upgrading from Legacy SynthPass

Transitioning to the new engine is automatic:

  • Trigger Cleanup: You no longer need to type + or - in the Length field. SynthPass 2 detects these, moves the data to the new Vault button, and cleans up the characters for you.
  • Classic Mode: If a password doesn't match a legacy one, check the Classic Mode box to use the old engine.

Verify Code Integrity (Audit)

SynthPass is source-available on GitHub so you can verify that the code running in your browser matches the developer's original build. This facilitates security auditing and trust.

Official Repository:
https://github.com/fruiz500/SynthPass

To verify the authenticity of the code after installation, do this:

1. Find your Extension Folder:
  • Mac: ~/Library/Application Support/Google/Chrome/Default/Extensions/khkpfnickpejcjhajmkljfadcaegphkd
  • Windows: %LOCALAPPDATA%\Google\Chrome\User Data\Default\Extensions\khkpfnickpejcjhajmkljfadcaegphkd
  • Linux: ~/.config/google-chrome/Default/Extensions/khkpfnickpejcjhajmkljfadcaegphkd
2. Download the Manifest:

Save the official hashes.txt file into that folder.

3. Run the Verification Command:
MacOS/Linux:
shasum -a 256 -c hashes.txt

Windows (PowerShell):
Get-Content hashes.txt | Where-Object { $_ -notmatch '^#' } | ForEach-Object { $h, $f = $_ -split ' +'; if ((Get-FileHash $f).Hash -eq $h) { "$f: OK" } else { "$f: FAIL" } }
Troubleshooting Mismatches (LF vs CRLF):
If a file says FAIL but you haven't changed the code, it is likely due to line-ending conversion. Windows often converts LF to CRLF, which changes the file's hash. To be certain, inspect the file manually via DevTools (F12) to audit the raw logic.
